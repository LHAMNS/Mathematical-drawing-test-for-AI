

你需要在 Desmos 里完成一个图形挑战，要求如下（非常严格）：

总体规则
	1.	只能提交一条表达式（单行）：我必须能把你给的内容 整行复制粘贴到 Desmos 里，立刻生成完整图形。
	•	不允许分成多条表达式
	•	不允许让我逐行粘贴
	•	不允许给多段代码/多行/多个对象
	2.	全程只允许 2D（x,y），但要实现“看起来像 3D 的透视效果”。
	3.	不能使用互联网搜索；如果你需要验证正确性，只能自己运行代码检查并自行修正直到正确。

图形：金字塔（2D 模拟 3D）

你要画出一个“金字塔”外观：在 2D 平面中呈现出 3D 立体透视效果。

视角要求
	•	默认视角：从斜上方看，整体约 45° 的立体效果（能看到至少两个面）。
	•	必须能通过一个可调变量 a 实现“摄像机绕金字塔中心做水平左右旋转（yaw）”：
	•	当旋转到背面时，必须真的能“看到反面”的结构（仍然是 2D 绘制，但视觉上要对应 3D 旋转后的结果）。
	•	旋转必须是整体刚体效果：所有细节随旋转正确变化。

缩放要求
	•	必须新增一个变量 C，用于控制整个金字塔的整体等比例缩放（包含所有细节一起缩放）。
	•	C 改变时不得产生纹理漂移或门位置变化错误。

⸻

门洞（只在正面）
	•	金字塔的“正面”必须有一个门洞（开口），且 只允许出现在正面：
	•	门洞 宽度=3，高度=2
	•	门洞位置必须在正面下方居中（具体位置以你定义的正面为准，但必须一致且正确）
	•	门洞内部必须是“空的”（不能被砖纹填充）
	•	当 a 旋转到背面时：
	•	门洞不能出现在背面或侧面
	•	不能因为旋转导致门洞漂移到错误位置

⸻

砖块纹理（仅填充实体部分）

在金字塔除门洞以外的实体部分内部，必须填充砖块纹理，要求：
	1.	砖块必须是横向排列的长方形（Running Bond 砌法）。
	2.	每块砖的尺寸必须精确为：
	•	宽 0.2
	•	高 0.1
	3.	砖块排列必须是错缝（Running Bond）：
	•	每一行相对上一行水平偏移 半块砖位 = 0.1
	4.	砖块纹理必须严格“贴在金字塔表面上”：
	•	调整 a 旋转时，砖块不能发生“漂移 / 滑动 / 错位”
	•	每块砖的空间位置要保持一致（随旋转只发生正确的刚体投影变化）
	5.	门洞区域内不得出现砖纹或砖缝线。

⸻

交付要求
	•	你最终只能输出 一条可粘贴进 Desmos 的单行表达式（包含 a 和 C 两个变量）。
	•	必须保证不会报错、不会出现除零或角度导致的崩坏。
	•	必须保证门洞尺寸、位置、只在正面出现这一点完全正确；砖块尺寸与错缝规则完全正确，并且随旋转/缩放不漂移。
